那么 sampler 不能把输出直接当 `ε`。必须先用

```text
x_0 = α_t x_t - σ_t v_t

```

## 🎯 主菜：先把“不确定”拆成两个来源

想象一辆自动驾驶汽车遇到两张图。

- 图 A：夜里下暴雨，车灯反光把行人边界冲得很模糊；
- 图 B：天气很好，但路边出现了一辆训练集中从未见过的怪异工程车。

两张图都可能让模型预测不稳，但原因不同：

```text
图 A：观测本身含糊                  -> aleatoric uncertainty
图 B：模型缺少相关知识或训练覆盖      -> epistemic uncertainty
```

中文里常分别叫：

- **偶然不确定性 / 数据不确定性**（aleatoric uncertainty）；
- **认知不确定性 / 模型不确定性**（epistemic uncertainty）。

最重要的区别是：增加同分布训练数据通常能降低 epistemic uncertainty，却不能消除样本本身不可辨认造成的 aleatoric uncertainty。

### 台阶 1：先从监督学习的概率问题写起

设：

- `x`：输入，例如 RGB、偏振图或特征；
- `y`：目标，例如深度、法向量或恢复后的像素；
- `D = {(x_i, y_i)}_{i=1}^N`：训练数据；
- `θ`：神经网络参数。

普通回归网络输出一个点估计：

```text
y_hat = f_θ(x)
```

若只用均方误差

```text
L_MSE = (y - f_θ(x))²
```

网络会告诉我们“我猜多少”，却没有直接告诉我们“这个猜测有多可靠”。

概率建模把问题改写为预测分布：

```text
p(y | x, D)
```

这个分布的宽窄就是不确定性的入口。但它的宽，可能来自两层随机性：

1. 即使知道正确模型参数 `θ`，同一个 `x` 对应的 `y` 仍有噪声；
2. 有限训练数据无法唯一确定参数 `θ`，多个合理模型会给出不同答案。

### 台阶 2：aleatoric uncertainty 是 `p(y | x, θ)` 里的散布

先假设参数 `θ` 已经固定。对回归任务，可以让网络输出均值 `μ_θ(x)` 和方差 `σ_θ²(x)`：

```text
y | x, θ ~ N(μ_θ(x), σ_θ²(x))
```

这里：

- `μ_θ(x)` 是模型认为最可能的目标；
- `σ_θ²(x)` 是在这个输入位置上，观测或标签本身有多含糊。

若所有样本共用同一个方差，叫 homoscedastic uncertainty；若方差随输入变化，叫 heteroscedastic uncertainty。图像中的阴影、反光、遮挡和低信噪比通常具有明显的空间差异，因此 dense prediction 更常关心后者。

为什么训练方差不能只让网络随便输出？从高斯似然完整推一遍。

高斯密度是：

```text
p(y | x, θ)
= 1 / sqrt(2πσ²) · exp(-(y - μ)² / (2σ²))
```

训练时最大化 likelihood，等价于最小化 negative log-likelihood（NLL）。先取对数：

```text
log p(y | x, θ)
= -1/2 log(2πσ²) - (y - μ)² / (2σ²)
```

再乘 `-1`：

```text
L_NLL
= (y - μ)² / (2σ²) + 1/2 log σ² + 1/2 log(2π)
```

最后一项与网络输出无关，可以省略：

```text
L_hetero = (y - μ)² / (2σ²) + 1/2 log σ²
```

这两个项互相制约：

- 残差大时，增大 `σ²` 会减小第一项，允许模型承认“这里确实难”；
- 但 `log σ²` 会惩罚无脑增大方差，防止模型把所有错误都解释成噪声。

所以 heteroscedastic NLL 不是“用 uncertainty mask 把难样本扔掉”，而是让误差大小与诚实报告的不确定性共同竞争。

工程上通常令网络输出 `s = log σ²`，则 `σ² = exp(s)`，损失写成：

```text
L_hetero = 1/2 exp(-s)(y - μ)² + 1/2 s
```

这样方差天然为正，也比直接预测一个接近零的 `σ²` 更稳定。

### 台阶 3：epistemic uncertainty 来自“哪些参数才对”

现在不再把 `θ` 当成唯一确定值。训练数据有限时，可能有很多组参数都能解释 `D`。贝叶斯写法是：

```text
p(θ | D) = p(D | θ)p(θ) / p(D)
```

其中：

- `p(θ)` 是训练前对参数的 prior；
- `p(D | θ)` 是这组参数解释训练数据的 likelihood；
- `p(θ | D)` 是看到数据后的 posterior。

对新输入 `x*`，不能只挑一组参数，而要把所有可能参数的预测加权平均：

```text
p(y* | x*, D)
= ∫ p(y* | x*, θ) p(θ | D) dθ
```

如果不同的合理 `θ` 在 `x*` 上给出相近答案，epistemic uncertainty 小；如果它们意见分裂，说明训练数据没有充分约束这个区域，epistemic uncertainty 大。

这就是为什么 out-of-distribution（OOD）输入常触发模型不确定性：模型在训练分布之外缺少证据，多个同样能拟合训练集的函数会向不同方向外推。

### 台阶 4：用全方差公式把两类不确定性真正分开

假设对于给定参数 `θ`，模型仍输出高斯分布：

```text
y | x, θ ~ N(μ_θ(x), σ_θ²(x))
```

现在 `θ` 本身服从 posterior `p(θ | D)`。预测方差满足 law of total variance：

```text
Var(y | x, D)
= E_{θ|D}[Var(y | x, θ)]
  + Var_{θ|D}(E[y | x, θ])
```

把条件均值和方差代进去：

```text
Var(y | x, D)
= E_{θ|D}[σ_θ²(x)]
  + Var_{θ|D}(μ_θ(x))
```

两项正好对应：

```text
aleatoric = 不同模型各自报告的数据方差的平均
epistemic = 不同模型预测均值之间的方差
```

这条分解非常关键，因为它给出了可计算的定义，而不只是两个名词。

实践中难以精确积分，常用 `M` 次近似模型采样，例如 deep ensembles、MC dropout 或近似贝叶斯网络。第 `m` 个模型输出 `μ_m, σ_m²`，则：

```text
预测均值：
μ_bar = (1/M) Σ_m μ_m

aleatoric：
U_alea = (1/M) Σ_m σ_m²

epistemic：
U_epi = (1/M) Σ_m (μ_m - μ_bar)²

总方差：
U_total = U_alea + U_epi
```

### 台阶 5：为什么“多给数据”只对其中一类有效

把两类不确定性放进三个场景：

#### 场景 A：标签仪器有随机噪声

如果传感器每次测量都带不可消除的抖动，同样的 `x` 会对应略不同的 `y`。收集更多相同质量的数据能更准确地估计噪声分布，却不能把噪声本身变成零。这主要是 aleatoric uncertainty。

#### 场景 B：训练集中没有透明材质

模型遇到透明物体时，不同初始化的模型给出截然不同的深度。补充覆盖透明材质的数据后，它们逐渐达成一致。这主要是 epistemic uncertainty。

#### 场景 C：图像既模糊又 OOD

现实中二者可以同时存在。模糊带来数据不确定性，未知类别带来模型不确定性。只输出一个“总 confidence”会把两种原因揉在一起，从而无法决定下一步该重拍图像、换传感器，还是补数据并重训。

### 台阶 6：不确定性不等于 error，也不自动等于 reliability

三个概念要分开：

- **error**：预测与真值实际差了多少，推理时通常未知；
- **uncertainty**：模型在没有真值时，对可能误差或预测分布宽度的估计；
- **reliability / calibration**：模型报告的 uncertainty 是否与真实错误频率匹配。

模型可能预测错却极度自信，这是低可靠性；也可能总是报告很大方差，虽然很少“过度自信”，却没有实际决策价值。

例如在回归中，如果模型给出的 95% prediction interval 真正覆盖约 95% 的测试目标，才说明区间具有较好的 coverage calibration。只看 uncertainty map 是否“长得像边缘”远远不够。

## 实现：最小 heteroscedastic regression + ensemble 分解

下面用多个网络近似参数 posterior。每个网络预测 `μ` 和 `log σ²`；同一输入上的模型内方差对应 aleatoric，模型间均值分歧对应 epistemic。

```python
import torch
import torch.nn as nn


class UncertaintyRegressor(nn.Module):
    def __init__(self, in_dim=1, hidden=64):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden),
            nn.SiLU(),
            nn.Linear(hidden, hidden),
            nn.SiLU(),
            nn.Linear(hidden, 2),
        )

    def forward(self, x):
        mean, log_var = self.net(x).chunk(2, dim=-1)
        log_var = log_var.clamp(-10.0, 6.0)
        return mean, log_var


def gaussian_nll(mean, log_var, target):
    # 0.5 * exp(-s) * residual^2 + 0.5 * s
    return 0.5 * (torch.exp(-log_var) * (target - mean).square() + log_var).mean()


@torch.no_grad()
def decompose_uncertainty(models, x):
    means, variances = [], []
    for model in models:
        mean, log_var = model(x)
        means.append(mean)
        variances.append(log_var.exp())

    means = torch.stack(means, dim=0)          # [M, B, 1]
    variances = torch.stack(variances, dim=0)  # [M, B, 1]

    predictive_mean = means.mean(dim=0)
    aleatoric = variances.mean(dim=0)
    epistemic = means.var(dim=0, unbiased=False)
    total = aleatoric + epistemic
    return predictive_mean, aleatoric, epistemic, total
```

真正训练 ensemble 时，各成员应使用不同随机初始化和数据顺序，必要时使用 bootstrap sampling。若所有成员从同一个 checkpoint 复制、看到完全相同的数据顺序，它们可能高度相关，从而低估 epistemic uncertainty。

## 🔗 与我 CVPR 工作的连接

偏振与 dense physical prediction 正好同时包含两类不确定性。

可能的 aleatoric 来源：

- 低 DoLP 区域的 AoLP 物理可观测性弱；
- 高光、饱和、运动或传感器噪声污染输入；
- 单张图像对某些几何存在多解；
- 标注或伪真值本身带误差。

可能的 epistemic 来源：

- 训练集中缺少某类材质、照明或几何；
- 合成到真实存在 domain gap；
- 模型容量或训练覆盖不足；
- 新数据落在训练分布之外。

因此，reliability-guided refiner 的可靠性图不能只靠“某个位置难不难”来解释。更严谨的实验可以是：

1. 用输入相关方差头估计 aleatoric uncertainty；
2. 用 ensemble 或 MC dropout 的预测分歧估计 epistemic uncertainty；
3. 分别与真实 angular error / reconstruction error 做相关性和 calibration 检查；
4. 在 low-DoLP、饱和、OOD 材质等子集分别报告结果；
5. 做 gating ablation：只用 aleatoric、只用 epistemic、两者联合。

一个尤其值得警惕的漏洞是循环论证：若 reliability map 直接由 ground-truth error 构造，训练和测试时的可用信息并不一致。论文中必须明确 reliability 是由哪些推理时可观测量估计的。

## 一句话总结

> Aleatoric uncertainty 说明“这个观测本身就含糊”，epistemic uncertainty 说明“模型在这里还缺知识”。前者位于给定模型后的数据分布 `p(y|x,θ)` 中，后者来自参数 posterior `p(θ|D)`；总预测方差可以由全方差公式拆成模型内方差与模型间分歧。

## 🔧 配菜·面试考点

**Q：为什么 heteroscedastic Gaussian NLL 不会让网络把方差无限预测大？**

**A：** 损失是 `0.5 exp(-s)(y-μ)^2 + 0.5s`，其中 `s=log σ²`。增大方差会降低残差项的权重，但同时线性增大 `0.5s`；二者形成约束。网络可以对难样本承认更大噪声，却要为这种不确定性付出代价。
