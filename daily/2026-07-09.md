## ⏮ 上篇回顾（30 秒）

B6 讲了 Latent Diffusion：

```text
image x -> VAE encoder -> latent z
latent z_t 上做 diffusion
z_0 -> VAE decoder -> image
```

它解决的是“每一步算得太贵”的问题。

今天解决另一个问题：

> 如果一步一步从纯噪声走回图像要走 1000 步，能不能少走几步？

答案是可以，但不是随便跳。采样加速的本质，是用更聪明的数值方法或训练目标，在少步数下尽量沿着正确的反向轨迹走。

## 🎯 主菜：采样加速

### 台阶 1：慢在哪里

DDPM 的采样可以写成：

```text
x_T -> x_{T-1} -> x_{T-2} -> ... -> x_0
```

每一步都要调用一次 U-Net：

```text
ε_θ(x_t, t)
```

如果 `T = 1000`，就意味着生成一张图要跑 1000 次 U-Net。U-Net 又很大，所以采样慢不是因为公式复杂，而是因为网络前向次数太多。

减少采样时间有两个方向：

1. 每次 U-Net 更便宜：比如 B6 的 latent diffusion；
2. U-Net 调用次数更少：今天的采样加速。

### 台阶 2：为什么不能直接从 `x_T` 跳到 `x_0`

直觉上，你可能会问：既然模型最终要还原干净图像，为什么不一步预测 `x_0`？

原因是扩散采样不是普通回归。`x_T` 接近纯噪声，里面几乎没有确定的图像结构。一开始模型只能给出很粗的方向；随着噪声变少，结构逐渐显出来，模型才能做更精细的修正。

如果步子太大，会出现三个问题：

- 方向误差被放大；
- 细节来不及逐步修正；
- 多模态分布被过早压成单一猜测。

所以少步数采样的核心矛盾是：

```text
步数越少，越快；
步子越大，越容易偏离真实反向轨迹。
```

### 台阶 3：把 DDIM 看成解一个 ODE

B3 讲过，扩散模型可以从 SDE / ODE 角度统一理解。DDIM 的确定性采样特别适合这个视角：它近似在解一条 probability-flow ODE。

先定义一个连续时间变量 `t`，`x(t)` 表示噪声水平为 `t` 时的样本。反向采样可以理解成从大噪声时间 `T` 往小噪声时间 `0` 积分：

```text
dx / dt = f_θ(x,t)
```

这里：

- `x` 是当前带噪样本；
- `t` 是噪声时间；
- `f_θ(x,t)` 是由噪声预测网络 `ε_θ` 诱导出来的速度场；
- 解这个 ODE，就是沿着反向轨迹从噪声走向数据。

普通 DDIM 可以粗略看成一阶数值积分：每次用当前点的方向走一步。步子小，误差小；步子大，误差大。

这就引出 DPM-Solver。

### 台阶 4：DPM-Solver 的直觉：用更高阶的积分方法少走弯路

DPM-Solver 不是重新训练一个模型，而是更聪明地使用已经训练好的 score / noise predictor。

它的想法可以简化成：

> 既然采样是在解一个 ODE，那就不要只用最粗的一阶 Euler 近似；可以利用更多时间点的信息，构造更高阶的数值解法。

一阶方法像这样：

```text
x_{t_next} = x_t + step_size * 当前方向
```

它只看当前方向。

二阶或三阶方法会额外估计方向如何变化。例如：

```text
先看 t 点方向
再看中间点或前一段方向
综合多个方向更新
```

这样同样走一大步时，轨迹更接近真实曲线。

你可以把它理解成开车过弯：

- 一阶方法：只看车头当前朝向；
- 高阶方法：还估计弯道曲率；
- DPM-Solver：利用扩散 ODE 的特殊形式，把这个估计做得更适合 diffusion。

### 台阶 5：DPM-Solver 为什么常常能十几步出图

扩散采样的反向轨迹不是完全任意的。噪声 schedule、score 形式、DDIM/ODE 结构都给了它规律。DPM-Solver 利用这些规律，把一部分可以解析处理的项单独拿出来，再对真正依赖神经网络的部分做高阶近似。

所以它比“随便少取几个 DDIM timestep”更稳。

但要注意：DPM-Solver 依然需要调用原来的模型。它省的是步数，不是每步网络成本。比如从 50 步变 20 步，本质是少跑 30 次 U-Net。

### 台阶 6：Consistency Models 的直觉：直接学“任意噪声到干净结果”的一致映射

DPM-Solver 是改 sampler。Consistency Models 更进一步：改训练目标。

它希望模型学到一个函数：

```text
f_θ(x_t, t) -> x_0
```

也就是从任意噪声水平的样本，直接映射到同一个干净结果。

关键是“一致性”。如果 `x_t` 和 `x_s` 在同一条反向轨迹上，它们应该对应同一个 `x_0`：

```text
f_θ(x_t, t) ≈ f_θ(x_s, s)
```

其中：

- `t` 和 `s` 是两个不同噪声时间；
- `x_t`、`x_s` 是同一条轨迹上的不同噪声状态；
- `f_θ` 的输出应当一致。

这种训练会鼓励模型在很多噪声水平上都能“看见终点”。采样时就可以用很少几步，甚至一步生成。

代价是：训练更复杂，质量和多样性要仔细权衡。一句话，一步生成很诱人，但让一步结果同时清晰、多样、可控，并不容易。

### 台阶 7：Distillation 的直觉：让学生模型模仿慢老师

Distillation 是另一条常见路线。

先有一个慢但强的 teacher sampler，比如 50 步或 100 步。再训练一个 student，让它用更少步数模仿 teacher 的结果。

可以写成：

```text
teacher: x_t -> 多步 -> x_s
student: x_t -> 一步 -> x_s
```

训练目标是让：

```text
student(x_t,t,s) ≈ teacher_transition(x_t,t,s)
```

如果反复把两步蒸馏成一步，就能把原来的多步采样逐渐压短。

这种方法的优势是直观、有效；代价是训练成本高，而且 student 可能继承 teacher 的偏差，还可能在极少步时损失多样性。

### 台阶 8：为什么少步数会牺牲质量

采样加速不是免费午餐。少步数常见问题包括：

- 纹理变糊；
- 局部结构不稳定；
- prompt / condition 跟随变差；
- 细节一致性下降；
- dense prediction 里的边界或物理量对齐变差。

根本原因是：扩散反向过程需要逐步纠错。步数越少，每一步承担的修正越多。只要模型方向、数值积分或蒸馏目标有一点偏差，都会被放大。

对普通文生图，轻微纹理问题还能接受；对 depth、normal、DoLP、AoLP 这类 dense physical modality，少步数误差可能直接破坏物理结构。

## 最小可运行代码：DDIM 子序列采样骨架

下面代码不实现完整 DPM-Solver，只展示“少取 timestep、减少 U-Net 调用次数”的基本骨架。真实 DPM-Solver 会使用更高阶更新公式。

```python
import torch


@torch.no_grad()
def ddim_step(x_t, t, t_next, eps_model, alpha_bar):
    eps = eps_model(x_t, t)

    a_t = alpha_bar[t].view(-1, 1, 1, 1)
    a_next = alpha_bar[t_next].view(-1, 1, 1, 1)

    x0_hat = (x_t - (1.0 - a_t).sqrt() * eps) / a_t.sqrt()
    x_next = a_next.sqrt() * x0_hat + (1.0 - a_next).sqrt() * eps
    return x_next


@torch.no_grad()
def sample_with_subsequence(x_T, eps_model, alpha_bar, timesteps):
    x = x_T
    for i in range(len(timesteps) - 1):
        t = torch.full((x.shape[0],), timesteps[i], device=x.device, dtype=torch.long)
        t_next = torch.full((x.shape[0],), timesteps[i + 1], device=x.device, dtype=torch.long)
        x = ddim_step(x, t, t_next, eps_model, alpha_bar)
    return x
```

这里的重点是：

```text
timesteps = [999, 949, 899, ..., 0]
```

如果只取 20 个时间点，就只调用 19 次模型。DPM-Solver、consistency、distillation 都是在这个目标上做得更系统：少调用模型，同时尽量不偏离高质量轨迹。

## 🔗 与你 CVPR 工作的连接

如果你的方向涉及偏振表征生成或 diffusion-based dense prediction，采样加速不能只看 FID 或肉眼图像质量。

你需要额外问：

- DoLP 边界是否更糊；
- AoLP 的角度周期性是否被破坏；
- `cos(2θ), sin(2θ)` 的单位圆结构是否保持；
- 低 DoLP 区域的不确定性是否被错误锐化；
- 少步数结果是否仍和 RGB / normal / depth 对齐。

论文实验里可以把 sampler 当成一个可靠性变量：

```text
same model + different sampler steps
compare perceptual quality + physical consistency + dense alignment
```

一句可以写进方法讨论的话：

> For physical dense prediction, fast diffusion sampling should be evaluated not only by perceptual fidelity but also by geometry-aligned and physics-aligned consistency under reduced denoising steps.

## 一句话总结

> 采样加速的核心，是把扩散反向过程从“上千次小步纠错”改成“更聪明的少步更新”。DPM-Solver 用更高阶 ODE 求解器少走弯路；Consistency Models 学任意噪声水平到干净结果的一致映射；Distillation 让少步 student 模仿多步 teacher。它们都能加速，但少步数会放大误差，尤其在偏振、深度、法向这类 dense physical task 上，必须检查物理一致性是否被牺牲。

## 🔧 配菜·面试考点

**Q：DDIM、DPM-Solver、Consistency Models 的区别是什么？**

**A：** DDIM 把采样变成确定性跳步过程，可以少取 timestep；DPM-Solver 把反向过程看成 ODE，用更高阶数值方法减少步数；Consistency Models 则改变训练目标，让模型学会从不同噪声水平直接映射到一致的干净结果。前两者更偏 sampler，后者更偏模型训练范式。

追问预备：为什么少步采样会损失质量；为什么 DPM-Solver 不需要重训；distillation 为什么能把多步压成少步。
