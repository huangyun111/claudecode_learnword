# 2026-07-07 · Latent Diffusion：为什么 Stable Diffusion 不在像素里直接扩散

> 阶段 B · 第 6 篇 ｜ 标签：diffusion, latent diffusion, Stable Diffusion, VAE, U-Net
> 读法：接着 B5。前面讲了 guidance 和条件注入，今天看 Stable Diffusion 里一个更底层的工程选择：为什么扩散过程不直接发生在 RGB 像素空间，而要先把图像压进 VAE latent。

## ⏮ 上篇回顾（30 秒）

B5 讲的是条件怎样进入 U-Net：

- 类别条件可以加到 time embedding；
- 文本条件常走 cross-attention；
- edge / depth / normal / DoLP 这类空间条件，更适合 ControlNet 或 adapter 的多尺度注入。

这些都默认有一个去噪网络：

```text
ε_θ(x_t, t, y)
```

但 Stable Diffusion 里，网络看到的通常不是像素图 `x_t`，而是 latent：

```text
ε_θ(z_t, t, y)
```

今天就回答这个问题：

> 为什么要把图像先压缩成 `z`，再在 `z` 上做 diffusion？

---

## 🎯 主菜：Latent Diffusion / Stable Diffusion

### 台阶 1：像素空间 diffusion 的成本从哪里来

先定义一张 RGB 图像：

```text
x ∈ R^{3×H×W}
```

如果分辨率是 `512×512`，那么像素空间张量大小是：

```text
3 × 512 × 512 = 786432
```

DDPM / DDIM / SDE 这些方法都要在这个空间里反复预测噪声。U-Net 的中间 feature 还会有很多通道，比如 `320, 640, 1280`。所以成本不是只和输入像素数有关，而是和多层 feature map 的面积一起增长。

如果直接在像素空间做高分辨率扩散，代价会很大：

- 训练显存高；
- 采样速度慢；
- 大量计算花在局部纹理和像素冗余上；
- 文本条件、cross-attention、多尺度控制都会进一步抬高成本。

直觉上，像素图里有很多相邻像素高度相关。天空里相邻两个蓝色像素、墙面里相邻纹理，并不都需要在高维空间里一粒一粒地扩散。

### 台阶 2：VAE 先学一个压缩空间

Latent Diffusion 的第一步，是训练一个 autoencoder，通常是 VAE 风格：

```text
z = E(x)
x_hat = D(z)
```

其中：

- `E` 是 encoder，把图像 `x` 压成 latent `z`；
- `D` 是 decoder，把 latent `z` 还原成图像 `x_hat`；
- `z` 的空间分辨率通常比原图小很多；
- `x_hat` 不需要逐像素完美等于 `x`，但要保留语义结构和可感知细节。

Stable Diffusion 常见设置里，`512×512` 图像会被压到大约：

```text
z ∈ R^{4×64×64}
```

大小是：

```text
4 × 64 × 64 = 16384
```

和像素空间的 `786432` 相比，维度大约小了：

```text
786432 / 16384 = 48
```

注意这不是说整体计算必然刚好快 48 倍，因为 U-Net 通道数、attention、实现细节都会影响速度。但它说明一个关键事实：扩散模型的主战场被搬到了更小的空间。

### 台阶 3：为什么这个 latent 不是普通压缩文件

不要把 `z` 理解成 JPEG 那种压缩码流。它更像一个可学习的连续表征空间。

VAE 的目标不是只追求最小文件体积，而是让 `z` 同时满足三件事：

1. 能被 decoder 还原成自然图像；
2. 空间尺寸足够小，便于 diffusion 训练；
3. 分布比较规整，便于加噪、去噪和采样。

一个简化版 VAE 训练目标可以写成：

```text
L_VAE = L_rec(x, D(z)) + λ_KL · KL(q_E(z|x) || N(0,I)) + λ_perc · L_perc(x, D(z))
```

逐项解释：

- `L_rec`：重建损失，让 `x_hat = D(z)` 和原图接近；
- `KL(q_E(z|x) || N(0,I))`：让 encoder 产生的 latent 分布不要太乱，接近标准高斯；
- `L_perc`：perceptual loss，让重建结果在视觉感知上接近原图；
- `λ_KL, λ_perc`：权重，控制不同目标的重要性。

工程上还可能加入 adversarial loss，让重建图更锐利。核心是：先得到一个“图像感知上有意义、维度又低”的 latent 空间。

### 台阶 4：扩散发生在 `z` 上，而不是 `x` 上

有了 VAE 后，训练 diffusion 的对象从图像 `x_0` 变成 latent `z_0`：

```text
z_0 = E(x)
```

前向加噪：

```text
z_t = sqrt(α_bar_t) z_0 + sqrt(1 - α_bar_t) ε
```

这里：

- `t` 是噪声时间步；
- `α_bar_t` 是到第 `t` 步累计保留的信号比例；
- `ε ~ N(0,I)` 是标准高斯噪声；
- `z_t` 是加噪后的 latent。

训练目标仍然可以是预测噪声：

```text
L = E_{z_0,t,ε,y} || ε - ε_θ(z_t,t,y) ||^2
```

这和 DDPM 的形式几乎一样，只是变量从 `x` 换成了 `z`。

采样时流程是：

```text
随机噪声 z_T
    ↓ diffusion reverse process
干净 latent z_0
    ↓ VAE decoder D
图像 x_hat
```

也就是：

```text
x_hat = D(z_0)
```

### 台阶 5：Stable Diffusion 的三件套

Stable Diffusion 可以粗略拆成三块：

```text
Text Encoder：prompt -> text tokens
Latent U-Net：在 z_t 上预测噪声
VAE Decoder：z_0 -> image
```

更完整一点：

1. 文本 prompt 进入 text encoder，得到条件 token `c`；
2. 随机 latent 噪声 `z_T` 进入 U-Net；
3. U-Net 在每个时间步用 cross-attention 读取文本条件 `c`；
4. sampler 根据噪声预测逐步得到 `z_0`；
5. VAE decoder 把 `z_0` 解码成 RGB 图像。

公式上可以写成：

```text
ε_θ(z_t, t, c)
```

这里的 `c` 是文本条件。B5 讲的 cross-attention、ControlNet、adapter，都是围绕这个 latent U-Net 的 feature flow 做文章。

### 台阶 6：latent 空间的收益

Latent Diffusion 的收益主要有四个。

第一，省显存和算力。

U-Net 不再处理 `3×512×512` 的像素，而处理 `4×64×64` 的 latent。高分辨率生成变得可训练、可部署。

第二，扩散模型更专注语义和结构。

VAE 已经承担了一部分低级像素建模工作。diffusion 在 latent 上学的是更压缩的图像因素：布局、形状、纹理摘要、语义组合。

第三，条件控制更容易集成。

文本 cross-attention、ControlNet、T2I-Adapter 都可以进入 latent U-Net。控制发生在压缩空间，但 decoder 仍能把结果还原成图像。

第四，工程生态更可扩展。

LoRA、DreamBooth、ControlNet、inpainting、image-to-image，很多后续方法都围绕这套 latent U-Net 改。它比从头训练像素扩散便宜太多。

### 台阶 7：latent 空间的代价

Latent Diffusion 不是白捡的。

第一，VAE 会丢信息。

如果 VAE 编码时丢掉了某些高频细节，diffusion 后面很难凭空精确恢复。比如文字、小图标、极细纹理，常常受 VAE 和 U-Net 两边共同限制。

第二，decoder 会带来自己的偏差。

最终图像一定经过 `D(z)`，所以 decoder 的表达能力会影响颜色、纹理和局部清晰度。

第三，对 dense prediction 任务要小心。

如果你要生成 depth、normal、DoLP、AoLP 这类物理或几何表征，latent 压缩可能会损伤像素级对齐。此时不能只问“latent 更省算力吗”，还要问：

> 这个 latent 是否保留了任务需要的局部物理信息？

这对你的偏振方向尤其重要。偏振编码不是普通 RGB 纹理，AoLP 的角度周期性、DoLP 的可靠性、边界附近的物理不确定，都可能在普通图像 VAE 里被压坏。

### 最小可运行代码：latent diffusion 的训练骨架

下面不是完整 Stable Diffusion，只展示“先编码成 latent，再对 latent 加噪训练”的核心流程。

```python
import torch
import torch.nn.functional as F


def q_sample(z0, t, noise, alpha_bar):
    """
    z0: clean latent [B,C,H,W]
    t: timestep indices [B]
    noise: Gaussian noise with same shape as z0
    alpha_bar: cumulative product schedule [T]
    """
    a = alpha_bar[t].view(-1, 1, 1, 1)
    return a.sqrt() * z0 + (1.0 - a).sqrt() * noise


def latent_diffusion_loss(vae_encoder, unet, images, text_cond, t, alpha_bar):
    with torch.no_grad():
        z0 = vae_encoder(images)

    noise = torch.randn_like(z0)
    zt = q_sample(z0, t, noise, alpha_bar)
    pred_noise = unet(zt, t, text_cond)
    return F.mse_loss(pred_noise, noise)
```

真实 Stable Diffusion 还会有 latent scaling factor、复杂 sampler、text encoder、CFG、EMA、mixed precision 等细节。但主逻辑就是这几行：

```text
image -> VAE latent -> add noise -> U-Net predicts noise -> decode latent
```

### 🔗 与你 CVPR 工作的连接

如果你要做偏振相关生成或 dense prediction，Latent Diffusion 给你两个方向。

第一，可以利用 latent diffusion 的高效生成能力，把偏振条件作为控制信号注入 latent U-Net：

```text
ε_θ(z_t, t, RGB, DoLP, AoLP, reliability)
```

这里重点不是简单 concat，而是设计物理条件如何进入多尺度特征。

第二，也要警惕普通 VAE latent 对物理量不友好。对偏振任务，你可能需要：

- 任务专用 VAE，让 latent 保留 DoLP/AoLP 结构；
- 对 AoLP 使用 `cos(2θ), sin(2θ)` 避免角度跳变；
- 对低 DoLP 区域引入 reliability mask；
- 在 decoder 或 loss 里加入物理一致性约束；
- 比较 pixel-space diffusion 和 latent-space diffusion 的边界保真度。

论文里可以这样说：

> Latent diffusion reduces the computational burden of diffusion-based dense prediction, but physical modalities require latent spaces that preserve geometry-aligned and physics-aligned signals rather than only perceptual image quality.

这句话的重点是：LDM 很强，但偏振不是普通图像生成，latent 的设计必须为物理表征服务。

### 一句话总结

> Latent Diffusion 的核心，是先用 VAE 把图像压到更小、更有语义的 latent 空间，再在这个空间里做 DDPM/DDIM 式去噪，最后用 decoder 还原图像。它让 Stable Diffusion 级别的高分辨率生成变得便宜，但代价是 latent 可能丢掉像素级或物理级细节；做偏振、深度、法向这类 dense task 时，必须检查 latent 是否保留了任务真正需要的信息。

## 🔧 配菜·面试考点

**Q：Stable Diffusion 为什么比像素空间 diffusion 更省？**

**A：** 因为它先用 VAE 把 `512×512×3` 的图像压成类似 `64×64×4` 的 latent，扩散 U-Net 的主要计算都发生在低分辨率 latent 上，而不是高分辨率像素上。这样显存、训练成本和采样成本都大幅下降。最后再用 VAE decoder 把干净 latent 还原成图像。

追问预备：VAE 为什么可能影响细节；LDM 和 pixel diffusion 的优缺点；为什么 dense physical modality 不能盲目使用普通图像 VAE。
