# 2026-07-06 · 条件注入机制：条件不是贴标签，而是进入 U-Net 的特征流

> 阶段 B · 第 5 篇 ｜ 标签：diffusion, conditional generation, cross-attention, ControlNet, T2I-Adapter, U-Net
> 读法：接着 B4。上一篇讲 guidance 是“采样方向怎么被条件改写”；今天看更底层的问题：条件 `y` 到底怎样进入网络，才能真的影响每一步去噪？

## ⏮ 上篇回顾（30 秒）

B4 讲过，条件生成的核心不是“多写一句 prompt”，而是把采样方向从：

```text
∇ log p_t(x)
```

改成：

```text
∇ log p_t(x | y)
```

Classifier guidance 用外部分类器给出条件梯度；classifier-free guidance（CFG）用：

```text
ε_cfg = ε_uncond + w(ε_cond - ε_uncond)
```

把“条件预测”和“无条件预测”的差放大。

但这里藏着一个前提：模型必须能算出 `ε_cond = ε_θ(x_t,t,y)`。也就是说，条件 `y` 必须进入网络。

今天就回答这个问题：

> 文本、边缘、深度、法向、偏振先验这些条件，究竟是怎样进入 U-Net 的？

---

## 🎯 主菜：条件注入机制

### 台阶 1：先定义“注入”到底是什么意思

先把符号摆清楚：

- `x_t`：第 `t` 个噪声水平下的带噪图像；
- `t`：时间步或噪声水平；
- `y`：条件，可以是文本、类别、RGB 图、边缘图、深度图、normal map、DoLP/AoLP 等；
- `ε_θ(x_t,t,y)`：带条件的噪声预测网络；
- `h`：U-Net 中间某一层的 feature map，形状通常是 `[B,C,H,W]`；
- `c`：条件编码后的 feature 或 token。

所谓条件注入，就是让网络在预测噪声时，不只看 `x_t` 和 `t`，还要在中间特征 `h` 里接收到 `y` 的信息。

最粗暴的方式是拼接：

```text
input = concat(x_t, y)
```

如果 `y` 是一张和图像对齐的条件图，比如 edge/depth/normal，这样有时可行。但它不够通用：

- 文本不是图像，不能直接按通道拼；
- 高层语义条件不一定和像素一一对齐；
- 太早拼进去，深层网络可能把条件冲淡；
- 条件很复杂时，单次拼接很难控制多尺度结构。

所以现代 conditional diffusion 更常见的是：条件在多个层级、多种方式里进入 U-Net。

### 台阶 2：类别条件最简单：加到时间嵌入里

类别条件 `y` 是一个 label，比如 “cat” 或 “dog”。它可以被 embedding 成一个向量：

```text
e_y = Embedding(y)
```

时间步 `t` 也会被编码成时间向量：

```text
e_t = TimeEmbedding(t)
```

最简单的做法是：

```text
e = e_t + e_y
```

然后把 `e` 注入每个 residual block。

直觉是：时间嵌入告诉网络“现在噪声有多大”；类别嵌入告诉网络“你应该往哪个类别去噪”。二者都像全局控制信号，会调节整张图的生成方向。

但它有明显限制：类别 embedding 是一个全局向量，适合告诉模型“生成什么类”，不适合告诉模型“左边是树、右边是人、边界在这里”。

### 台阶 3：文本条件为什么常用 cross-attention

文本 prompt 会先经过 text encoder，变成一串 token embedding：

```text
c = [c_1, c_2, ..., c_N]
```

其中 `N` 是 token 数量。U-Net 中某层图像特征 `h` 可以看成很多空间位置的向量：

```text
h = [h_1, h_2, ..., h_M]
```

这里 `M = H × W`，每个 `h_i` 对应一个空间位置。

Cross-attention 的问题是：

> 对图像中的每个位置 `h_i`，它应该关注文本里的哪些 token？

标准 attention 公式是：

```text
Q = h W_Q
K = c W_K
V = c W_V
Attention(h,c) = softmax(QK^T / √d) V
```

逐项解释：

- `Q`（query）来自图像特征，表示“图像当前位置想问什么”；
- `K`（key）来自文本 token，表示“每个词能被怎样匹配”；
- `V`（value）也来自文本 token，表示“匹配后真正拿走的信息”；
- `d` 是 key/query 的维度，用 `√d` 缩放避免点积过大；
- `softmax(QK^T / √d)` 是图像位置对文本 token 的注意力权重。

所以 cross-attention 不是把文本贴在图像旁边，而是让每个图像位置都能动态读取文本 token。比如 prompt 里有 “red car”，生成车身区域的 feature 更可能关注 `red` 和 `car`，天空区域则关注别的 token 或弱关注。

### 台阶 4：为什么 cross-attention 能控制语义，但不擅长精确结构

Cross-attention 很适合文本，因为文本是离散 token，提供的是语义约束。它告诉模型：

- 画面里有什么；
- 大致风格是什么；
- 对象之间有什么关系。

但如果你要求“边缘必须贴着这条线”“深度必须符合这张图”“偏振角结构必须保持”，单靠文本 cross-attention 就不够。

原因是：文本 token 没有天然的像素对齐关系。它可以强语义，但弱几何。对于 dense prediction 或物理表征生成，我们往往需要空间对齐条件。

这就引出 ControlNet 和 T2I-Adapter。

### 台阶 5：ControlNet 的核心：复制一条可训练的控制分支

ControlNet 的直觉很工程：

> 原来的大 diffusion U-Net 已经会生成图像，不要把它改坏；另外复制一条分支专门读条件图，把控制信号逐层加回主干。

设原始 U-Net 某层特征是：

```text
h_l
```

条件图 `y` 经过控制分支后得到同尺度控制特征：

```text
g_l(y)
```

然后在主干对应层加入：

```text
h_l' = h_l + Z_l(g_l(y))
```

这里 `Z_l` 常用 zero convolution，也就是初始权重为 0 的卷积层。

为什么要 zero conv？因为训练一开始：

```text
Z_l(g_l(y)) ≈ 0
```

模型行为几乎等于原始预训练 U-Net，不会突然被随机控制分支破坏。随着训练推进，控制分支逐渐学会在各层加入有用的结构信号。

这点非常重要：ControlNet 的强，不只是“多了一个条件输入”，而是它保护了预训练生成能力，同时给空间条件开了一条可训练通道。

### 台阶 6：T2I-Adapter 的核心：轻量条件适配器

T2I-Adapter 的思路比 ControlNet 更轻：

> 不复制完整 U-Net，而是训练一个小 adapter，把条件图编码成多尺度 feature，再注入预训练模型。

可以写成：

```text
[a_1, a_2, ..., a_L] = Adapter(y)
h_l' = h_l + a_l
```

其中 `a_l` 是和 U-Net 第 `l` 层尺度对应的条件特征。

它的优点：

- 参数少，训练成本低；
- 可以为不同条件类型训练不同 adapter；
- 更适合快速接入 edge、depth、pose、sketch 等结构条件。

它的代价：

- 控制能力通常弱于完整 ControlNet；
- 对复杂条件或强约束任务，可能不如 ControlNet 稳。

你可以把它们的差别简单记成：

```text
ControlNet：更重，更强，复制控制分支。
T2I-Adapter：更轻，更便宜，提供多尺度条件补丁。
```

### 台阶 7：对视觉条件，关键是“尺度”和“对齐”

如果条件是文本，重点是语义对齐；如果条件是图像，重点是空间对齐。

比如：

- edge map：控制边界；
- depth map：控制前后层次；
- normal map：控制局部表面方向；
- segmentation map：控制区域类别；
- DoLP/AoLP：控制偏振强度和角度结构。

这些条件不是一句全局描述，而是每个位置都有信息。因此它们应该在多个分辨率进入 U-Net：

```text
高分辨率层：边缘、纹理、细节位置
中分辨率层：局部结构、物体部件
低分辨率层：全局布局、语义关系
```

这就是多尺度注入的意义。条件只在输入层拼一次，容易被后续下采样和非线性冲淡；多层注入则能在不同尺度持续提醒网络：“结构在这里，不要跑偏。”

### 最小可运行代码：一个轻量条件 adapter

下面是一个极简版 adapter，不是完整 ControlNet，但能体现“条件图变多尺度特征，再加到 U-Net 中间层”的思路。

```python
import torch
import torch.nn as nn


class TinyConditionAdapter(nn.Module):
    def __init__(self, cond_channels=1, widths=(64, 128, 256)):
        super().__init__()
        layers = []
        in_ch = cond_channels
        for width in widths:
            layers.append(
                nn.Sequential(
                    nn.Conv2d(in_ch, width, 3, padding=1),
                    nn.SiLU(),
                    nn.Conv2d(width, width, 3, padding=1),
                    nn.SiLU(),
                )
            )
            in_ch = width
        self.layers = nn.ModuleList(layers)
        self.down = nn.AvgPool2d(2)

    def forward(self, cond):
        feats = []
        x = cond
        for layer in self.layers:
            x = layer(x)
            feats.append(x)
            x = self.down(x)
        return feats


class ToyUNetBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.block = nn.Sequential(
            nn.Conv2d(channels, channels, 3, padding=1),
            nn.SiLU(),
            nn.Conv2d(channels, channels, 3, padding=1),
        )

    def forward(self, h, cond_feat=None):
        if cond_feat is not None:
            h = h + cond_feat
        return self.block(h)


B, H, W = 2, 64, 64
condition = torch.randn(B, 1, H, W)      # e.g. edge/depth/DoLP map
adapter = TinyConditionAdapter(1)
cond_feats = adapter(condition)

print([f.shape for f in cond_feats])
```

真实工程里，你需要让 `cond_feats[l]` 的通道数和 U-Net 对应层匹配，必要时加 `1×1 conv` 做投影。ControlNet 还会有更完整的 encoder/decoder 控制分支和 zero conv。

### 🔗 与你 CVPR 工作的连接

如果你的任务是从 RGB 或其他输入生成偏振表征，那么“条件注入机制”比“换一个 sampler”更接近方法核心。

你可以把条件分成三类：

1. **语义条件**：物体类别、场景类别、材质类别；
2. **几何条件**：depth、normal、edge、mask；
3. **物理条件**：DoLP、AoLP、偏振编码、可靠性或不确定性图。

对应设计思路：

- 语义条件适合 embedding 或 cross-attention；
- 几何条件适合 ControlNet / Adapter 的多尺度空间注入；
- 物理条件不应只当普通图像通道，最好配合物理含义设计 loss、mask 或 reliability guidance。

论文里可以把方法讲成：

> 我们不是把偏振先验简单拼接到输入，而是在扩散 U-Net 的多尺度特征层注入物理对齐条件，使反向去噪过程在每个尺度同时受到图像结构和偏振物理约束。

这句话比“we concatenate polarization prior with RGB”强很多，因为它指出了控制发生的位置：不是输入端一次性拼接，而是反向生成过程的多尺度特征流。

### 一句话总结

> Guidance 解决“采样方向怎么被条件拉动”，条件注入解决“模型怎样看见条件”。类别条件可以加到时间嵌入，文本条件常用 cross-attention，空间条件更适合 ControlNet 或 T2I-Adapter 这类多尺度特征注入；对于偏振、深度、法向等 dense condition，关键是对齐、尺度和物理含义。

## 🔧 配菜·面试考点

**Q：ControlNet 和普通通道拼接有什么区别？**

**A：** 普通拼接只在输入端把条件图和 `x_t` 合在一起，后续网络是否保留条件信息完全靠自己学。ControlNet 会复制一条控制分支，把条件编码成多尺度控制特征，并通过 zero conv 逐层加回预训练 U-Net。这样既保留原模型的生成能力，又能在不同分辨率持续施加空间约束。

追问预备：zero conv 为什么能稳定训练；cross-attention 为什么适合文本；T2I-Adapter 为什么比 ControlNet 轻。
