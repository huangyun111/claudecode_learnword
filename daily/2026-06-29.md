## ⏮ 昨日回顾（30 秒）

B1 我们啃下了 DDPM：网络学的是"看着带噪图 `x_t`、猜出噪声 `ε`"（`L_simple`），采样时**从纯噪声一步步去噪**。但有个硬伤——**慢**：训练用的 `T` 通常是 1000，采样就得老老实实跑 1000 步。今天解决它。

---

## 🎯 主菜：DDIM

### 台阶 1：DDPM 到底慢在哪

DDPM 的前向是一条**马尔可夫链**：`x_0 → x_1 → … → x_T`，每一步只依赖前一步。反向采样也就被锁死成"必须一步一步退"——你不能从 `x_{1000}` 直接跳到 `x_{500}`，因为反向公式 `p_θ(x_{t-1}|x_t)` 只定义了"退一格"。1000 格就得走 1000 次网络前向。**这是采样慢的根**。

### 台阶 2：一个被忽略的"松动处"

回看 B1 的训练目标：

```
L_simple = E ‖ ε − ε_θ( √(ᾱ_t)·x_0 + √(1-ᾱ_t)·ε , t ) ‖²
```

**它只用到了边缘分布 `q(x_t|x_0) = N(√ᾱ_t·x_0, (1-ᾱ_t)I)`**——也就是"任意 t 一步加噪"那条闭式。训练**从头到尾没用到那条马尔可夫链的逐步结构**。

换句话说：训练出来的 `ε_θ(x_t, t)` 本质是一个**"任意噪声水平下的去噪器"**——给它任意 noise level `t` 的带噪图，它就吐出噪声估计。它**并不在乎**当初是怎么一格一格加上去的。

> 这就是 DDIM 的全部底气：**网络绑定的是边缘 `q(x_t|x_0)`，不是那条马尔可夫链。** 那我们就有自由换一条"走法"。

### 台阶 3：DDIM 的主意——换一条路，但保持每个路口的"风景"不变

DDIM 构造了一族**非马尔可夫**的前向过程 `q_σ`，要求：**每个 t 的边缘仍然是 `q(x_t|x_0)=N(√ᾱ_t·x_0,(1-ᾱ_t)I)`，和 DDPM 一模一样**。边缘不变 ⟹ 同一个 `ε_θ` 原封不动继续能用、不用重训。

它直接**定义**那个带 `x_0` 条件的后验（注意这是设计出来的，不是推出来的）：

```
q_σ(x_{t-1}|x_t, x_0) = N( √(ᾱ_{t-1})·x_0  +  √(1-ᾱ_{t-1}-σ_t²)·(x_t − √ᾱ_t·x_0)/√(1-ᾱ_t) ,  σ_t²·I )
```

逐项读：
- `√(ᾱ_{t-1})·x_0`：把干净图缩放到 `t-1` 的信号水平。
- 中间那项 `(x_t − √ᾱ_t·x_0)/√(1-ᾱ_t)`：由 (★) 闭式可知，这一坨**正好等于当前的归一化噪声方向**；前面乘 `√(1-ᾱ_{t-1}-σ_t²)` 把它放到 `t-1` 该有的噪声量。
- `σ_t²`：**留给自己调的随机性旋钮**（台阶 5 细说）。

DDIM 论文证明了：这样定义，边缘确实保持 `q(x_t|x_0)` 不变（这是它的核心引理，结论拿来用即可）。

### 台阶 4：反向采样公式——"先猜干净图，再重新合成"

采样时我们手上只有 `x_t`，没有 `x_0`。沿用 B1 的招——**用 `ε_θ` 反解出对 `x_0` 的估计**：

```
x̂_0 = ( x_t − √(1-ᾱ_t)·ε_θ(x_t,t) ) / √(ᾱ_t)
```

把 `x̂_0` 代回台阶 3 的均值（并注意中间那项的噪声方向就是 `ε_θ`），得到 DDIM 的一步更新：

```
x_{t-1} = √(ᾱ_{t-1})·x̂_0  +  √(1-ᾱ_{t-1}-σ_t²)·ε_θ(x_t,t)  +  σ_t·z ,   z~N(0,I)
          └──── 指向干净图 ────┘   └──── 重新加回该有的噪声 ────┘   └ 随机项 ┘
```

直觉：**每一步都先看穿"这张图去噪后大概长啥样"(`x̂_0`)，再把它打回 `t-1` 的噪声水平。**

### 台阶 5：那个旋钮 σ_t —— 一头连 DDPM，一头连确定性

- **`σ_t = 0`** ⟹ 随机项消失，更新**完全确定性**：`x_{t-1}=√ᾱ_{t-1}·x̂_0 + √(1-ᾱ_{t-1})·ε_θ`。**这就是狭义的 DDIM**。给定 `x_T`，输出唯一确定。
- **`σ_t = √(β̃_t)`**（DDPM 那个后验方差）⟹ 退回**原版 DDPM** 的随机采样。

常用一个参数 `η∈[0,1]` 把两端连起来：`σ_t = η·√(β̃_t)`。**η=0 是 DDIM，η=1 是 DDPM**，中间随意调。一个旋钮统一了两者。

### 台阶 6：为什么能"跳步"加速

关键：台阶 3 的 `q_σ` 对**任意一对 `(t, 上一个时刻)`** 都成立，并不要求相邻。于是可以只取一个**子序列** `τ_1 < τ_2 < … < τ_S`（比如从 1000 步里挑 50 步），在子序列上跑同样的更新式：

```
从 x_{τ_i} 一步到 x_{τ_{i-1}}，公式同台阶 4（把 t→τ_i, t-1→τ_{i-1}）
```

`S` 可以远小于 `T`（50 步常常就够）。**网络没变、不用重训，纯推理阶段提速 20–50×。**

### 台阶 7：确定性白送的三个红利

`σ=0` 时采样是确定性的 `x_T → x_0` 映射，于是：
1. **可逆**：能反过来把真实图 `x_0` 编码回它对应的 `x_T`（DDIM inversion），用于真实图像编辑。
2. **隐空间插值有语义**：在两张图的 `x_T` 之间线性插值，解码出平滑的语义过渡。
3. **通往 ODE**：这个确定性过程其实是一条 **probability-flow ODE** 的离散化——这正是下一篇 B3（score/SDE 统一框架）要接的口子。

### 实现：DDIM 采样（PyTorch 伪代码）

```python
@torch.no_grad()
def ddim_sample(model, shape, abar, timesteps, eta=0.0):  # timesteps: 降序子序列, 如 [999,979,...,0]
    x = torch.randn(shape, device=device)
    for i, t in enumerate(timesteps):
        t_prev = timesteps[i + 1] if i + 1 < len(timesteps) else -1
        at      = abar[t]
        at_prev = abar[t_prev] if t_prev >= 0 else torch.tensor(1.0, device=device)
        eps = model(x, t)
        x0  = (x - (1 - at).sqrt() * eps) / at.sqrt()                 # 台阶4：猜干净图
        sigma = eta * (((1 - at_prev) / (1 - at)) * (1 - at / at_prev)).sqrt()
        x = at_prev.sqrt() * x0 + (1 - at_prev - sigma**2).sqrt() * eps \
            + sigma * (torch.randn_like(x) if t_prev >= 0 else 0)     # 台阶4更新式
    return x
```

### 🔗 与你 CVPR 工作的连接

- **推理提速**：你的偏振表征生成（conditional diffusion）如果用 DDPM 跑 1000 步，做大规模评测/可视化会很肉。换 DDIM 50 步，几乎不掉质量却快几十倍——直接救你的 ablation 与对比实验效率。
- **确定性 = 可复现**：`η=0` 下"同一 RGB 条件 + 同一初始噪声 → 唯一偏振输出"。这对你那套**评测指标与可视化对比体系**是刚需：要公平比较两个方法/两个 setting，必须消除采样随机性，DDIM 给你确定性输出。
- **DDIM inversion** 还能让你把真实样本编码回 latent，分析你的条件生成在隐空间里如何被 RGB 条件"推动"。

### 一句话总结

> DDIM 看穿了"网络只认边缘分布"，于是换一条非马尔可夫的路、保持边缘不变 → 同一个 `ε_θ` 能**跳步**采样；`σ=0` 时还白送确定性、可逆与通往 ODE 的门票。

## 🔧 配菜·面试考点

**Q：** DDIM 既然能几十步出图，为什么还能用 DDPM 训练好的同一个网络？它俩到底什么关系？

**A：** 因为 **DDPM 的训练目标只依赖边缘 `q(x_t|x_0)`，与马尔可夫逐步结构无关**——`ε_θ` 学到的是"任意噪声水平的去噪器"。DDIM 构造了一族非马尔可夫前向，**刻意保持每个 t 的边缘与 DDPM 相同**，所以网络无需重训即可复用；同时它的反向后验里多了个方差旋钮 `σ`(用 `η` 参数化)：`η=1` 退回 DDPM 随机采样、`η=0` 是确定性 DDIM。确定性让它可跳子序列采样从而提速，并带来可逆性。
↳ 追问预备：`η=0` 与 probability-flow ODE 的关系；为什么少步数下 DDIM 比 DDPM 质量更稳；DPM-Solver 如何在此基础上进一步用高阶 ODE 解法加速（B7）。
