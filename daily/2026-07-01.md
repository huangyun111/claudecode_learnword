# 2026-07-01 · Score/SDE：把 DDPM 和 DDIM 收进同一张地图

## ⏮ 昨日回顾（30 秒）

B1 里，DDPM 训练网络预测噪声 `ε`；B2 里，DDIM 发现网络其实只依赖边缘分布 `q(x_t|x_0)`，所以可以换一条非马尔可夫的采样路，还能跳步。

今天把这两件事收进一个更大的框架：

> 扩散模型不是只能理解成"一步步去噪"。更本质地看，它是在每个噪声水平上学习一个 **score**，也就是"概率密度往哪里变大"的方向；采样就是沿着这个方向，把纯噪声推回数据分布。

## 🎯 主菜：Score-based / SDE 统一框架

### 台阶 1：先把 score 说成人话

假设某个噪声水平 `t` 下，图像随机变量叫 `x_t`，它有一个概率密度 `p_t(x)`。

这里：

- `x`：一张图像，数学上可以看成一个很高维向量；
- `p_t(x)`：在噪声水平 `t` 时，`x` 这张图出现的概率密度；
- `log p_t(x)`：概率密度的对数；
- `∇_x log p_t(x)`：对图像 `x` 求梯度，得到一个和 `x` 同形状的向量。

这个向量就叫 **score**：

```
s_t(x) = ∇_x log p_t(x)
```

直觉是什么？

`log p_t(x)` 越大，说明这个 `x` 越像当前噪声水平下的"正常样本"；`∇_x log p_t(x)` 指向让 `log p_t(x)` 上升最快的方向。所以 score 就是：

> 在当前噪声水平下，如果你想让这张图更像真实分布，应该往哪边挪。

如果 `x_t` 是一张乱糟糟的带噪图，score 告诉你"朝哪个方向动一点，会更像数据"。这就是去噪的方向感。

### 台阶 2：为什么预测噪声 `ε` 其实是在学 score

回到 DDPM 的闭式加噪：

```
x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε,    ε ~ N(0,I)
```

这等价于：

```
q(x_t | x_0) = N(√ᾱ_t · x_0, (1-ᾱ_t)I)
```

现在固定 `x_0`，把 `q(x_t|x_0)` 看成关于 `x_t` 的高斯密度。一个多维高斯：

```
N(μ, σ²I)
```

它的 log density 去掉常数项是：

```
log q(x) = - 1/(2σ²) · ||x - μ||² + const
```

对 `x` 求梯度：

```
∇_x log q(x) = - (x - μ) / σ²
```

代回 DDPM 里的：

```
μ = √ᾱ_t · x_0
σ² = 1 - ᾱ_t
```

得到：

```
∇_{x_t} log q(x_t | x_0)
= - (x_t - √ᾱ_t · x_0) / (1 - ᾱ_t)
```

再用闭式加噪式：

```
x_t - √ᾱ_t · x_0 = √(1-ᾱ_t) · ε
```

代进去：

```
∇_{x_t} log q(x_t | x_0)
= - √(1-ᾱ_t) · ε / (1 - ᾱ_t)
= - ε / √(1 - ᾱ_t)
```

也就是说：

```
ε = - √(1 - ᾱ_t) · ∇_{x_t} log q(x_t | x_0)
```

这一步很关键：**预测噪声 `ε`，只差一个已知缩放系数和负号，就等价于预测条件 score**。

那为什么我们最终想要的是 `∇ log p_t(x_t)`，而不是 `∇ log q(x_t|x_0)`？因为训练时 `x_0` 是从真实数据里采样的，网络看到大量 `(x_t,t)`，做 MSE 预测最优解时学到的是条件 score 在所有可能 `x_0` 上的平均效果。这个平均后的方向，正是当前噪声边缘分布 `p_t(x_t)` 的 score。直观说：

> 网络不是只记某一张干净图该怎么去噪，而是在学"所有真实图像被加到这个噪声水平后，整体概率密度往哪里更高"。

所以 B1 的 `ε_θ(x_t,t)` 换个写法就是：

```
s_θ(x_t,t) ≈ - ε_θ(x_t,t) / √(1 - ᾱ_t)
```

这里 `s_θ` 是神经网络预测的 score。

### 台阶 3：score matching 是什么

如果我们能直接拿到真实的 `∇_x log p_t(x)`，就可以训练：

```
E || s_θ(x_t,t) - ∇_{x_t} log p_t(x_t) ||²
```

这叫 **score matching**：让网络预测的 score 接近真实 score。

麻烦是：真实数据分布 `p_data(x)` 不知道密度，`p_t(x)` 也不知道密度，更别说它的梯度。

扩散模型的聪明处在于：我不知道真实分布的 score，但我知道"给真实样本加高斯噪声"这个条件分布 `q(x_t|x_0)` 的 score。于是用 **denoising score matching**：

```
E_{x_0, x_t} || s_θ(x_t,t) - ∇_{x_t} log q(x_t|x_0) ||²
```

把上一步推出来的式子代入，就变成预测噪声的 MSE：

```
E || ε - ε_θ(x_t,t) ||²
```

所以你现在可以把 B1 的训练目标重新翻译一遍：

> DDPM 不是神秘地让网络"猜噪声"。猜噪声只是一个方便的参数化；本质上，它是在做 denoising score matching，让网络学每个噪声水平下概率密度上升的方向。

### 台阶 4：从离散时间到连续时间

DDPM 用 `t=1,2,...,T` 这些离散步。Score-based / SDE 框架把时间变成连续的：

```
t ∈ [0,1]
```

其中：

- `t=0`：接近真实数据分布；
- `t=1`：接近标准高斯噪声；
- `x(t)`：连续时间下的随机图像状态。

前向加噪可以写成一个 **SDE**（stochastic differential equation，随机微分方程）：

```
dx = f(x,t) dt + g(t) dw
```

逐项解释：

- `dx`：从时间 `t` 到 `t+dt`，图像状态的微小变化；
- `f(x,t)`：drift，确定性漂移，决定平均往哪边走；
- `g(t)`：diffusion coefficient，噪声强度；
- `dw`：Wiener process 的微小增量，可以直觉理解成连续时间的高斯随机扰动；
- `dt`：非常小的时间间隔。

这句话的白话版：

> 前向过程一边按 `f` 漂移，一边按 `g` 往里撒随机噪声，最后把真实图像慢慢变成高斯噪声。

DDPM 就是这个连续过程的一种离散化版本。

### 台阶 5：反向 SDE：怎么从噪声走回数据

前向 SDE 把数据变噪声。神奇的是，如果知道每个时刻的 score：

```
∇_x log p_t(x)
```

就能写出反向时间的 SDE：

```
dx = [ f(x,t) - g(t)^2 ∇_x log p_t(x) ] dt + g(t) dŵ
```

这里的 `dŵ` 是反向时间里的随机项。注意这个式子里最重要的东西：

```
- g(t)^2 ∇_x log p_t(x)
```

它把 score 塞进了反向动力学里。直觉上：

- 前向：持续加噪，让图像远离数据分布；
- 反向：用 score 告诉系统"哪里更像数据"，再沿这个方向把噪声拉回来。

真实 score 不知道，就用网络：

```
∇_x log p_t(x) ≈ s_θ(x,t)
```

所以采样就变成：从 `x(1) ~ N(0,I)` 出发，数值求解反向 SDE，一步步走到 `x(0)`。

这就是 score-based generative modeling 的核心。

### 台阶 6：Probability-flow ODE：去掉随机项的确定性版本

同一个前向 SDE，还对应一条确定性的 **probability-flow ODE**：

```
dx = [ f(x,t) - 1/2 · g(t)^2 ∇_x log p_t(x) ] dt
```

它和反向 SDE 的差别：

- 反向 SDE：有随机项，采样路径会抖；
- probability-flow ODE：没有随机项，给定初始噪声，路径完全确定。

为什么它也能生成？因为这条 ODE 在每个时间 `t` 的边缘分布，和原来的 SDE 保持一致。也就是说：

> SDE 和 ODE 走的单条路径不同，但它们在每个时刻看到的整体样本分布是同一个。

这就把 B2 的 DDIM 接上了：`η=0` 的 DDIM 是确定性采样，给定 `x_T` 输出唯一；从连续视角看，它就是 probability-flow ODE 的一种离散近似。

### 台阶 7：DDPM、DDIM、SDE/ODE 的关系图

可以这样收纳：

```
前向加噪：
真实数据 x_0  --逐渐加噪-->  高斯噪声 x_T

训练目标：
预测 ε
≈ 预测 -√(1-ᾱ_t) · score
≈ 学 s_θ(x_t,t)

采样方式：
DDPM      = 反向 SDE 的离散随机采样
DDIM η=0 = probability-flow ODE 的离散确定性采样
DPM-Solver 等 = 更高阶地数值求解这个 ODE/SDE
```

这就是统一框架最有用的地方：它不再把 DDPM、DDIM、DPM-Solver 看成互不相关的技巧，而是看成同一条连续动力系统的不同求解方式。

### 最小可运行代码：从 `epsilon` 换成 `score`

```python
import torch

def eps_to_score(eps_pred, alpha_bar_t):
    """
    eps_pred: model predicted noise, shape [B,C,H,W]
    alpha_bar_t: cumulative product alpha_bar at t, shape [B] or scalar
    return: score estimate s_theta(x_t,t)
    """
    while alpha_bar_t.ndim < eps_pred.ndim:
        alpha_bar_t = alpha_bar_t.view(*alpha_bar_t.shape, 1)
    return -eps_pred / torch.sqrt(1.0 - alpha_bar_t)


def probability_flow_euler_step(x, t, dt, eps_model, alpha_bar_t, f, g):
    """
    One Euler step for a toy probability-flow ODE:
        dx = [f(x,t) - 0.5 * g(t)^2 * score] dt
    This is schematic code: real samplers need a specific noise schedule.
    """
    eps = eps_model(x, t)
    score = eps_to_score(eps, alpha_bar_t)
    drift = f(x, t) - 0.5 * (g(t) ** 2) * score
    return x + drift * dt
```

真正工程里你不一定显式写 `score`，很多库仍然让模型输出 `epsilon`、`v` 或 `x0`。但只要知道它们可以互相换，你看采样器就不会迷路。

### 🔗 与你 CVPR 工作的连接

你的偏振表征生成如果继续往 conditional diffusion 做，SDE/ODE 视角有三个直接用处：

1. **采样器选择不再玄学**：DDIM、DPM-Solver、Euler、Heun，本质是对同一个反向动力系统的不同数值解法。你可以把"步数/质量/稳定性"作为数值求解问题来调。
2. **条件控制更容易解释**：RGB 条件、边缘、法向、DoLP/AoLP 先验，本质是在改变 score 的方向，让样本往满足条件的高概率区域走。下一篇讲 guidance 时会直接用到这句话。
3. **可复现实验更稳**：ODE/DDIM 确定性采样适合 ablation；SDE 随机采样适合多样性。你可以按评测目的选择，而不是一股脑用默认 sampler。

### 一句话总结

> DDPM 的"预测噪声"本质上是在学 score；score 告诉样本往哪里更像数据。连续化之后，前向加噪是 SDE，反向生成可以走随机 SDE，也可以走确定性 probability-flow ODE；DDPM、DDIM 和各种快速采样器，都是这张地图上的不同路线。

## 🔧 配菜·面试考点

**Q：为什么说 DDPM 预测噪声等价于学习 score？**

**A：** 因为 DDPM 的闭式前向为 `x_t = √ᾱ_t x_0 + √(1-ᾱ_t) ε`，所以 `q(x_t|x_0)` 是均值 `√ᾱ_t x_0`、方差 `(1-ᾱ_t)I` 的高斯。高斯 log density 对 `x_t` 的梯度是 `-(x_t-√ᾱ_t x_0)/(1-ᾱ_t)`；再代入 `x_t-√ᾱ_t x_0 = √(1-ᾱ_t)ε`，得到 `∇ log q(x_t|x_0) = -ε/√(1-ᾱ_t)`。因此预测 `ε` 只差一个已知缩放，就等价于预测 denoising score。训练在数据分布上取期望后，网络学到的是当前噪声边缘分布的 score。

追问预备：反向 SDE 里为什么出现 `g(t)^2 score`；probability-flow ODE 为什么没有随机项还能保持同样边缘；DDIM 和 ODE sampler 的关系。
