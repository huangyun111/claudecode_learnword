# 每日知识推送 · 生成规程

本规程是「手动触发」与「云端自动」**共用的唯一内容生成逻辑**，保证两种入口产出一致、进度连贯。

## 输入：先读这些

1. `STATE.md` —— 当前阶段、下一篇主菜、难度基线、用户最新反馈、复习队列
2. `ROADMAP.md` —— 接下来该讲的 ☐ 条目
3. `INDEX.md` —— 已推送过什么（避免重复）
4. `daily/` 最近 1–2 篇 —— 衔接上文、做"昨日回顾"

## 步骤

1. **确定日期**：用今天的真实日期 `YYYY-MM-DD`（手动时取自环境 currentDate 或问用户；自动时取运行日）。
2. **读状态**：按上面"输入"读全。若 `STATE.md` 反馈区有新偏好（太深 / 太浅 / 换方向 / 赶实验），**优先据此调整**今天内容。
3. **定今天的菜单**：
   - 主菜 = ROADMAP 当前阶段的下一个 ☐ 条目（推进一步，**不跳步**）。
   - 配菜 = 一道呼应主菜的面试题 / 手撕点。
   - 前沿 = 仅每周 2–3 次（看星期与 STATE 上次记录）；其余日子省略，**不硬凑**。
4. **生成** `daily/YYYY-MM-DD.md`，用下方模板。
5. **更新进度**：
   - `ROADMAP.md`：讲完的条目 ☐ → ☑。
   - `STATE.md`：推进"下一篇主菜"、把今天主题加入"已讲清单"、按间隔规则更新"复习队列"、更新前沿"上次"日期。
   - `INDEX.md`：表格顶部加一行。
6. **手动模式收尾**：把今天内容讲给用户，并主动邀请："可以追问 / 要代码 / 说太浅太深 / 换方向。"

## 内容质量要求（重要）

- **直觉先行**：先讲清"为什么需要它、在解决什么问题"，再上数学。
- **数学严谨但克制**：给关键推导与最终结论，不堆砌中间步骤；标注每个符号含义。
- **最小可运行代码**：PyTorch 伪代码或精简实现，能说明核心机制即可。
- **必带两个连接**：
  - 🔗 **与我 CVPR 工作的连接**：概念如何关联偏振表征生成 / reliability-confidence 机制 / 深度法向估计。
  - 💼 **面试视角**：大厂会怎么考这个点，标准答法是什么。
- 难度按 `STATE.md` 基线：不从零讲，默认已懂扩散基础。

## daily 模板

```markdown
# {YYYY-MM-DD} · {主菜标题}

> 阶段 {X} · 第 {N} 篇 ｜ 标签：{tag1, tag2, ...}

## ⏮ 昨日回顾（30 秒）
（一两句勾连昨天；第一天写"开篇"）

## 🎯 主菜：{概念}
### 直觉：为什么需要它
### 数学：核心推导
### 实现：最小代码
### 🔗 与我 CVPR 工作的连接
### 一句话总结

## 🔧 配菜·面试考点
**Q：** ...
**A：** ...

## 📡 前沿速览（每周 2–3 次，其余日省略）
- 论文 / 动机 / 方法 / 与我的关系

## ✅ 今日打卡
- 今天掌握了：...
- 复习提醒：...
- 想深入可以问我：...
```
