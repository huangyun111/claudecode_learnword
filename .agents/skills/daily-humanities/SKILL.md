# 人文通识推送 · 生成规程

本规程是人文线「手动触发」与「云端自动」**共用的唯一内容生成逻辑**，保证两种入口产出一致、进度连贯。与技术线（`daily-knowledge`）完全独立、互不干扰。

## 输入：先读这些

1. `humanities/STATE.md` —— 当前模块、下一篇主线、难度基线、最新反馈、复习队列
2. `humanities/ROADMAP.md` —— 接下来该讲的 ☐ 条目
3. `humanities/INDEX.md` —— 已推送过什么（避免重复）
4. `humanities/posts/` 最近 1–2 篇 —— 衔接上文、做"上篇回顾"

## 步骤

1. **确定日期**：用今天的真实日期 `YYYY-MM-DD`（手动时取自环境 currentDate 或问用户；自动时取运行日）。
2. **读状态**：按上面"输入"读全。若 `STATE.md` 反馈区有新偏好（太散 / 换模块 / 多讲中国…），**优先据此调整**。
3. **定今天的主题**：
   - 主线 = `ROADMAP.md` 当前模块的下一个 ☐ 条目（推进一步，**不跳步**；除非用户要求跳模块）。
   - 涉及前置历史背景时，可临场简补（对应 H1 "按需回填"）。
4. **生成** `humanities/posts/YYYY-MM-DD.md`，用下方模板。
5. **更新进度**：
   - `humanities/ROADMAP.md`：讲完的条目 ☐ → ☑。
   - `humanities/STATE.md`：推进"下一篇主线"、把今天主题加入"已讲清单"、按规则更新"复习队列"。
   - `humanities/INDEX.md`：表格顶部加一行。
6. **手动模式收尾**：把今天内容讲给用户，并主动邀请："可以追问 / 要深入某点 / 说太散太浅 / 换模块。"

## 内容质量要求（重要）

- **先给画面 / 抛问题**：开篇用一个场景、一张地图、一个反差把人拽进来，再讲脉络。
- **脉络优先，不堆砌**：讲清因果链和关键转折，一篇只咬一条主线；宁深勿全，绝不写成"知识点罗列"。
- **必带两个落点**（对称于技术线的 CVPR 连接 + 面试视角）：
  - 🌍 **与当下的连接**：这段历史 / 地理如何解释今天的某条新闻、某个现象。
  - 💬 **谈资点 / 反常识洞见**：一个能在饭桌或面试上抛出的角度、金句或"原来如此"。
- **政治话题务必中立**：涉及争议（中美、俄乌、制度之争、宗教冲突…），**给事实 + 呈现各方视角和动机，不灌输单一结论**。目标是培养判断力，不是塞立场。这是底线。
- **清晰为本、语气自然**（核心基线）：以"清晰、有画面、不啰嗦"为底（参考技术线 DDPM 那篇的通透感），讲人话、不端着，像一个博学的朋友平静地把事讲明白，而不是念讲稿。可用**恰当的比喻**帮助理解（如"地理像河床""半部天书"），但比喻服务于"看懂"，**点到为止**。
- **不要刻意搞笑、不堆段子**（重要教训）：过度的生活化类比（小区邻居、投胎手气）、网络梗、抖机灵会让人**尴尬、出戏，反而看不进去**。趣味来自"把复杂的事讲通透"的爽感，不是来自段子。**宁可平实，不要油。**
- **干货与中立**：框架 + 洞见 + 与当下的连接一个不能少；政治话题给事实和多方视角、不灌输立场；沉重话题（战争 / 伤亡）正经讲。
- 难度按 `humanities/STATE.md` 基线：理科思维强、人文底子一般，讲透"为什么"，少要求"背"。

## post 模板

```markdown
# {YYYY-MM-DD} · {主题标题}

> 模块 {Hx} · 第 {N} 篇 ｜ 标签：{tag1, tag2, ...}

## ⏮ 上篇回顾（30 秒）
（一两句勾连上一篇；第一篇写"开篇"）

## 🎬 开篇：一个画面 / 一个问题
（场景、地图、反差，把人拽进来）

## 📖 主线：来龙去脉
（因果链与关键转折，可分几个小节，每节能独立站住）

## 🌍 与当下的连接
（这段知识怎么解释今天还在发生的事）

## 💬 谈资点 / 反常识洞见
（一个能复述、能抛出去的角度或金句）

## 一句话总结

## ✅ 今日打卡
- 今天理清了：...
- 复习提醒：...
- 想深入可以问我：...

（结尾一句预告下一篇）
```
